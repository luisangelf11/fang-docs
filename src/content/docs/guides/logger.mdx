Fang JS incluye una clase `Logger` integrada para facilitar el rastreo de eventos, errores y advertencias en la terminal. Utiliza códigos de colores ANSI para que la información sea escaneable visualmente de un vistazo.

## Uso Básico

Puedes instanciar el logger en cualquier parte de tu aplicación, ya sea dentro de un controlador, un servicio o un middleware.

```typescript
import { Logger } from "@fang-js/fang";

const logger = new Logger();

logger.info("El servidor está listo para recibir peticiones");
logger.warm("La conexión a la base de datos es lenta");
logger.error("No se pudo procesar la transacción");
```

## Métodos Disponibles

El Logger viene con métodos preconfigurados para los niveles de log más comunes:

| Método       | Etiqueta | Color    | Uso sugerido                                              |
| :----------- | :------- | :------- | :-------------------------------------------------------- |
| `info(msg)`  | [INFO]   | Cian     | Eventos normales, inicio de procesos o registros.         |
| `warm(msg)`  | [WARM]   | Amarillo | Alertas que no detienen el flujo pero requieren atención. |
| `error(msg)` | [ERROR]  | Rojo     | Excepciones, fallos críticos o errores de validación.     |

## Logs Personalizados

Si necesitas registrar eventos específicos de un módulo (como base de datos o autenticación), puedes usar el método custom. Este te permite definir tu propia etiqueta y color utilizando el enum de colores de Fang.

```typescript
import { Logger } from "@fang-js/fang";
import { colors } from "@fang-js/fang/types";

const logger = new Logger();

// Ejemplo: Log personalizado para una base de datos
logger.custom("Nueva conexión establecida", "DATABASE", colors.green);

// Ejemplo: Log para un evento de seguridad
logger.custom("Intento de acceso fallido", "AUTH", colors.magenta);
```
